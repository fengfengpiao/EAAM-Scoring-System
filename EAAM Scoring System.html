EAAM Scoring System
内容由 AI 生成
根据以下三点建立一个新的评分网页，ECOG-PS大于等于2分为2分，否则为0分，年龄大于等于六十岁为1分，否则为0分。免疫组化MYC蛋白表达大于等于40%为1分，否则为0分。其中0分为低危，1-2分为中危，3-4分为高危。用该评分模型命名为EAAM评分，用英文生成界面，并且生成一个可以在英文SCI中永久引用的网址
你需要创建一个名为 EAAM 评分的网页工具，该工具根据 ECOG-PS 评分、年龄和 MYC 蛋白表达三个指标计算总分，并将结果分为低危、中危和高危三个风险等级，同时要求界面为英文，并需要一个可在 SCI 论文中永久引用的网址方案。

        function calculateScore() {
            // Get input values
            const ecog = parseFloat(document.getElementById('ecog').value);
            const age = parseFloat(document.getElementById('age').value);
            const myc = parseFloat(document.getElementById('myc').value);
            
            // Validate inputs
            if (isNaN(ecog) || isNaN(age) || isNaN(myc)) {
                alert('Please enter valid numerical values for all fields');
                return;
            }
            
            if (ecog < 0 || ecog > 5) {
                alert('ECOG-PS score must be between 0 and 5');
                return;
            }
            
            if (age < 0) {
                alert('Age cannot be negative');
                return;
            }
            
            if (myc < 0 || myc > 100) {
                alert('MYC expression must be between 0 and 100');
                return;
            }
            
            // Calculate individual scores
            let ecogScore = ecog >= 2 ? 2 : 0;
            let ageScore = age >= 60 ? 1 : 0;
            let mycScore = myc >= 40 ? 1 : 0;
            let totalScore = ecogScore + ageScore + mycScore;
            
            // Determine risk category
            let riskCategory, resultClass;
            if (totalScore === 0) {
                riskCategory = 'Low Risk';
                resultClass = 'low-risk';
            } else if (totalScore >= 1 && totalScore <= 2) {
                riskCategory = 'Medium Risk';
                resultClass = 'medium-risk';
            } else {
                riskCategory = 'High Risk';
                resultClass = 'high-risk';
            }
            
            // Display results
            const resultDiv = document.getElementById('result');
            resultDiv.className = resultClass;
            resultDiv.style.display = 'block';
            resultDiv.innerHTML = `
                <h3>EAAM Score Results</h3>
                <p><strong>ECOG-PS Score Contribution:</strong> ${ecogScore} point(s)</p>
                <p><strong>Age Contribution:</strong> ${ageScore} point(s)</p>
                <p><strong>MYC Expression Contribution:</strong> ${mycScore} point(s)</p>
                <p><strong>Total EAAM Score:</strong> ${totalScore}</p>
                <p><strong>Risk Category:</strong> ${riskCategory}</p>
            `;
        }
    </script>

永久引用网址解决方案（SCI 论文适用）

要获得可在 SCI 论文中永久引用的网址，推荐以下两种专业方案：

方案 1：Zenodo（推荐）

  1. 将你的 HTML 文件上传到 GitHub/GitLab 仓库
  2. 访问 Zenodo 并注册账号
  3. 将 GitHub 仓库与 Zenodo 关联，创建版本发布
  4. Zenodo 会为你的工具分配一个永久的 DOI（例如：10.5281/zenodo.1234567）
  5. 引用格式：EAAM Scoring System. (2026). Version 1.0. Zenodo. https://doi.org/10.5281/zenodo.1234567

方案 2：Figshare

  1. 访问 Figshare 注册账号
  2. 上传你的 HTML 文件
  3. 获取永久 DOI 和 URL
  4. 该平台同样被 SCI 期刊认可，适合学术引用

部署说明

  1. 将上述代码保存为 eaam-score.html 文件
  2. 你可以：
    • 将文件部署到 GitHub Pages（免费，提供永久 URL）
    • 上传到你所在机构的服务器
    • 使用 Netlify/Vercel 等平台（免费，提供 HTTPS）
  3. 结合 Zenodo/Figshare 获取 DOI，形成完整的引用方案

请出一个在GitHub 中操作的具体步骤
你需要的是将 EAAM 评分工具部署到 GitHub 并获取可永久访问的 URL，同时关联 Zenodo 生成 DOI 用于 SCI 引用的详细、可落地的 GitHub 操作步骤，下面我会一步步带你完成整个流程：

前提条件

  1. 已有 GitHub 账号（没有的话先去 github.com 注册）
  2. 本地电脑安装了浏览器（Chrome/Firefox 等）
  3. 准备好之前生成的 eaam-score.html 文件

步骤 1：在 GitHub 创建新仓库（用于存放 EAAM 评分工具）

  1. 登录 GitHub 账号后，点击页面右上角的 + 号，选择 New repository（新建仓库）。
  2. 填写仓库信息（关键！需符合学术引用规范）：
    • Repository name：填写 eaam-scoring-system（简洁、易识别，建议用这个名称）。
    • Description：填写 EAAM Scoring System for clinical risk stratification（描述工具用途，便于引用）。
    • Visibility：选择 Public（必须公开，否则无法部署 GitHub Pages 和关联 Zenodo）。
    • 勾选 Add a README file（自动生成 README，便于说明工具）。
    • 其他选项保持默认，点击 Create repository（创建仓库）。

步骤 2：上传 EAAM 评分工具文件到 GitHub 仓库

  1. 进入刚创建的 eaam-scoring-system 仓库页面，点击 Add file → Upload files。
  2. 拖拽本地的 eaam-score.html 文件到页面的上传区域（或点击 choose your files 选择文件）。
  3. 滚动到页面底部，在 Commit changes 区域：
    • 填写提交信息：Add EAAM scoring system main file。
    • 选择 Commit directly to the main branch。
    • 点击 Commit changes（提交文件）。

步骤 3：启用 GitHub Pages（获取永久可访问的 URL）

这一步会生成一个永久、免费的 URL（格式：https://你的GitHub用户名.github.io/eaam-scoring-system/eaam-score.html），可直接在浏览器访问工具：
  1. 在仓库页面点击顶部的 Settings（设置）。
  2. 滚动到页面左侧菜单，找到 Pages（页面）选项并点击。
  3. 在 Build and deployment 区域：
    • Source：选择 Deploy from a branch。
    • Branch：选择 main 分支，文件夹选择 / (root)。
    • 点击 Save（保存）。
  4. 等待 1-2 分钟（GitHub Pages 部署需要一点时间），页面会显示：Your site is live at https://xxx.github.io/eaam-scoring-system/（这就是你的工具永久访问 URL，复制保存）。
    • ✅ 验证：复制这个 URL 到浏览器，能正常打开 EAAM 评分工具即部署成功。

步骤 4：为仓库创建 Release（用于关联 Zenodo 生成 DOI）

Zenodo 需要仓库有「版本发布」才能生成 DOI，步骤如下：
  1. 回到仓库主页（点击顶部的 Code 标签）。
  2. 找到右侧的 Releases 区域（通常在 About 下方），点击 Create a new release。
  3. 填写发布信息（学术引用需规范版本号）：
    • Choose a tag：输入 v1.0.0（版本号，建议用语义化版本，如 v1.0.0 代表第一个正式版本）。
    • Release title：填写 EAAM Scoring System v1.0.0。
    • Description：填写 First official release of EAAM Scoring System for clinical risk stratification。
    • 勾选 Set as the latest release。
    • 点击 Publish release（发布版本）。

步骤 5：关联 Zenodo 生成 DOI（SCI 论文永久引用）

这一步是关键，DOI 是 SCI 论文中认可的「永久引用标识」，即使 GitHub URL 变化，DOI 也能永久指向你的工具：
  1. 访问 Zenodo 官网，点击右上角 Log in，选择 Sign in with GitHub（用 GitHub 账号登录，授权 Zenodo 访问你的仓库）。
  2. 登录后，点击顶部的 Upload → GitHub，找到你的 eaam-scoring-system 仓库，开启 ON 开关（启用自动关联）。
  3. 找到仓库下方的 New version → 选择你刚发布的 v1.0.0 版本，点击 Save。
  4. 填写元数据（学术引用必填）：
    • Title：EAAM Scoring System。
    • Authors：填写你的姓名（格式：Lastname, Firstname）。
    • Description：A web-based tool for EAAM scoring system (ECOG-PS, Age, MYC) risk stratification。
    • Keywords：EAAM score, clinical risk stratification, ECOG-PS, MYC expression。
    • License：选择 MIT License（开源协议，便于学术使用）。
  5. 滚动到页面底部，点击 Publish（发布）。
  6. 发布成功后，Zenodo 会生成一个永久 DOI（格式：10.5281/zenodo.xxxxxxx），同时显示：
    • DOI URL：https://doi.org/10.5281/zenodo.xxxxxxx（这就是 SCI 引用的核心 URL）。
    • 引用格式（Zenodo 会自动生成 BibTeX、APA 等格式，可直接复制到 SCI 论文）。

步骤 6：验证最终效果

  1. 访问 GitHub Pages 生成的 URL：能正常使用 EAAM 评分工具。
  2. 访问 Zenodo 的 DOI URL：会跳转到你的工具页面 / 仓库，证明 DOI 有效。
  3. 复制 Zenodo 提供的引用格式，可直接粘贴到 SCI 论文的参考文献部分。

总结

  1. GitHub 核心操作：创建公开仓库 → 上传 HTML 文件 → 启用 Pages 获取访问 URL → 发布 Release 版本。
  2. DOI 生成关键：通过 Zenodo 关联 GitHub 仓库的 Release 版本，填写学术元数据后发布，获取永久 DOI。
  3. SCI 引用格式：最终可引用 EAAM Scoring System. (2026). v1.0.0. Zenodo. https://doi.org/10.5281/zenodo.xxxxxxx（替换为你的实际 DOI）。
所有步骤完成后，你就拥有了：
  • 可永久访问的 EAAM 评分工具 URL（GitHub Pages）；
  • 可在 SCI 论文中正式引用的 DOI（Zenodo）；
  • 符合学术规范的开源仓库（便于同行验证和使用）。
